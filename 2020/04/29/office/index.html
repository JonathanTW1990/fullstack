通过docker安装nextcloud＋onlyoffice＋mysql并配置ip地址ssl

主要内容

  1. 通过docker安装nextcloud＋onlyoffice＋mysql
  2. 配置通过https://192.168.9.51 登陆(其中192.168.9.51为演示IP地址，你需要修改为你实际IP地址)
  3. 配置 SSL 自签名
  4. 最终实现通过https://192.168.9.51 来访问服务器，避免出现ssl不可信的情况

    系统要求

  • 系统: Centos7
  • 内存: 2G 以上
  • 硬盘: 10G 以上

    系统初始化

    关闭selinux，自动更新时间，停止firewalld

    1、安装程序[install processing]

    打开浏览器输入服务器IP地址，http://your_ip_address 在出现的配置页面中输入

    • 用户名：admin，管理密码
    • 数据库选择mysql，数据库的帐号、密码、数据库名、数据库地址信息，见docker-compose.yml 文件
    • 注意：配置的数据库地址localhost 为mariadb

配置完成之后，开启onlyoffice支持 [when u finish config webpage, u can use command support onluoffice support]

打开浏览器http://your_ip_address 通过上面配置的admin帐号和密码进行登陆
如果不需要通过域名访问，或者https访问，到此配置成功，您可以使用了[All processing down if u don’t need use doman to access or https]

文件位置说明 [document path]

  • nginx配置文件[nginx config path]
    /root/docker-onlyoffice-nextcloud-mysql/nginx.conf
  • nextcloud数据存储位置[nextcloud config path]
    /var/lib/docker/volumes/dockeronlyofficenextcloudmysql_app_data/_data

以上将搭建好 http 方式的nextcloud ，如果想通过https 访问，请继续下面的教程 [top all commands are for http access,if u need https keep going below command]

制作自签名证书 [make cert by yourself]

会出现如下提示,根据提示输入省份、组织、邮箱地址等[it show some prompt , depend on prompt u need to type some information like address]

退出 app-server docker ，并拷贝生成的证书到本地 [exit , and copy cert to local path]

  • 修改docker-compose 文件，映射本地证书文件夹到docker 的nginx下[edit docker-compose document , mapping local cert folder to nginx of docker]
  • 在nginx 的volumes 下面增加目录映射，如下所示最后一行[for example u can add last example command under docker-compose.yml > volumes: for mapping of path]
  • 修改nginx 配置文件，配置ssl证书[go to edit nginx config to config ssl cert]

    （1）在server部分做如下改动，ssl 开头的三行内容是增加的[u can find server part command,u can see our example ,three of tittle of ssl_ you need to add by yourself]

（2）然后在倒数第二行增加如下内容目的是为了访问http 的时候自动跳转到https[and then add this two command for that when someone access http it will auto go to https]

  • 修改 nextcloud 配置文件[edit nextcloud config]

    修改其中的30、43行，为你的nextcloud 访问地址https://192.168.9.51 [edit line 30 and 43 for ur access address for example:]
    array (
    0 => ‘192.168.9.51’,
    1 => ‘nginx-server’,
    ),
    ‘datadirectory’ => ‘/var/www/html/data’,
    ‘dbtype’ => ‘mysql’,
    ‘version’ => ‘15.0.0.10’,
    ‘overwrite.cli.url’ => ‘https://192.168.9.51',

修改onlyoffice 的default.json 文件，信任不可信的ssl [edit onlyoffice default.json document for trust or not trust ssl]

修改”rejectUnauthorized”: true 为”rejectUnauthorized”: false[edit command rejectUnauthorized”: true to “rejectUnauthorized”: false]

在onlyoffice-document-server 的volumes 部分增加最后一行 [go to add one line command in end of volumes: part in docker-compose.yml]

  • 重启docker-compose[reopen docker-compose]

  • 注意:用docker-compose restart 是不能加载分区的 *

  • 通过浏览器登陆 nextcloud 修改onlyoffice 配置[use Browser to login extcloud and change onlyoffice config]

    到设置中选择onlyoffice，打开onlyoffice api 页面 [go to setting find onlyoffice api and turn open]
    把Server address for internet requests form the Document Editing Src，修改为你的nextcloud 访问地址https://192.168.9.51[change Server address for internet requests form the Document Editing Src to ur access address]
    最后点击保存[save!]
